# 新手先看：项目目录、配置文件和发布流程

如果你刚接触这个项目，最容易懵的不是代码，而是：

- 我到底该改哪个文件
- 为什么我改了内容，页面没变化
- 本地预览和线上发布分别怎么做

这篇就是整个系列的开场说明，建议先看完，再去看后面的具体功能文章。

## 先记住这 4 句话

1. 这个项目真正读取的文章目录是 `src/content/posts/`
2. 大部分功能配置都在 `src/config/` 里
3. 本地预览用 `pnpm dev`
4. 推送上线用 `.\publish-blog.bat`

只要先记住这四句，后面基本就不容易乱。

## 这个项目最常改的目录

### 1. 文章目录

```text
src/content/posts/
```

你以后写文章、改文章，主要就是这里。

比如：

```text
src/content/posts/17.md
src/content/posts/22.md
```

### 2. 配置目录

```text
src/config/
```

这里放的是站点配置，比如：

- 网站标题
- 导航栏
- 公告
- 评论系统
- 音乐播放器
- 看板娘
- 侧边栏
- 相册

### 3. 公共静态资源目录

```text
public/
```

这里通常放：

- 图标
- 音乐
- 公开图片
- 赞赏码
- 相册图片

### 4. 特殊页面内容目录

```text
src/content/spec/
```

这里一般不是普通文章，而是：

- 关于页内容
- 留言板说明

## 为什么有时候你改了文件，网站却没变化

最常见原因有这几个：

### 1. 你改错目录了

比如你改的是：

```text
src/content/17.md
```

但项目真正用的是：

```text
src/content/posts/17.md
```

所以以后优先改 `src/content/posts/`。

### 2. 本地服务没刷新

可以先试：

1. 保存文件
2. 刷新浏览器
3. 不行就重启一次 `pnpm dev`

### 3. 你本地改了，但还没推送

本地看到变化，不等于 GitHub Pages 已经更新。

## 小白最常用的 3 个命令

### 1. 本地预览

```powershell
cd E:\GITHUB\firefly-replica
pnpm dev
```

作用：

- 启动本地博客
- 你改完文件后可以立刻预览

### 2. 构建检查

```powershell
cd E:\GITHUB\firefly-replica
pnpm build
```

作用：

- 检查有没有明显报错
- 确认站点能不能正常生成

### 3. 发布上线

```powershell
cd E:\GITHUB\firefly-replica
.\publish-blog.bat
```

作用：

- 自动 `git add`
- 自动 `git commit`
- 自动 `git push`

如果你之前已经提交过，只是推送失败了，就用：

```powershell
cd E:\GITHUB\firefly-replica
.\push-only.bat
```

## 推荐你按这个顺序学

如果你是第一次接触这个项目，建议按下面的顺序看：

1. 先看部署和基础信息配置
2. 再看首页外观相关内容
3. 然后看评论、音乐、看板娘这些功能
4. 接着看写文章相关内容
5. 最后看 SEO、备份、排错这些进阶内容

这样会比乱跳着看更容易上手。

## 这一整套系列会教你什么

看完这套文章后，你应该能做到：

- 自己部署博客
- 修改标题、导航、页脚和首页
- 配好评论、音乐、看板娘、公告
- 会写 Markdown 文章
- 会插图、插视频、管理分类标签
- 会用置顶、加密、分享海报、版权协议
- 会开留言板、赞助页、相册页
- 知道 RSS、SEO、备份和排错怎么做

## 新手最后一定要知道的提醒

### 1. 不要一上来同时改很多功能

最稳的方法是一次只改一个点，改完就刷新看效果。

### 2. 能本地先看，就不要直接盲推

本地预览能帮你省掉很多低级错误。

### 3. 文章写作和项目配置是两回事

- 写文章：主要改 `src/content/posts/`
- 改功能：主要改 `src/config/`

### 4. 看不懂代码没关系，先会改配置就够了

你现在的目标不是“会开发这个主题”，而是“会把它用起来”。
