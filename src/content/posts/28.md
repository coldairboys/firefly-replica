# Hermes Agent 完整安装部署指南

Hermes Agent 是 Nous Research 推出的一款可成长的 AI Agent，主打自主学习、多层记忆架构和丰富的工具系统。这篇指南将带你从零开始完成安装部署。

## 项目简介

| 项目 | 说明 |
|------|------|
| 开发团队 | Nous Research |
| 开源协议 | MIT（完全免费） |
| 核心卖点 | 闭环学习系统 + 四层记忆架构 + 40+ 内置工具 |
| 支持平台 | Linux、macOS、WSL2、Android（Termux） |
| 消息平台 | Telegram、Discord、Slack、WhatsApp、飞书 |
| 支持模型 | Claude、OpenRouter、DeepSeek、Qwen、Ollama 本地模型等 |

**官方资源：**
- 官网：https://hermes-agent.nousresearch.com
- GitHub：https://github.com/NousResearch/hermes-agent

## 一、系统要求

### 1.1 硬件配置

| 配置项 | 最低要求 | 推荐配置 |
|--------|----------|----------|
| CPU | 2 核 | 4 核以上 |

- ✅ **Windows**（需通过 WSL2）

## 二、环境准备

### 2.1 安装 Git

Git 是唯一需要手动安装的依赖，其他依赖都会由安装脚本自动处理。

**Linux（Ubuntu/Debian）：**
```bash
sudo apt update
sudo apt install git -y
```

**Linux（CentOS/RHEL）：**
```bash
sudo yum install git -y
```

**macOS：**
macOS 通常自带 Git，可通过以下命令验证：
```bash
git --version
```

如果没有安装，会提示安装 Xcode Command Line Tools，按提示操作即可。

**Windows（WSL2）：**
先安装 WSL2，然后在 WSL2 中安装 Git：
```bash
sudo apt update
sudo apt install git -y
```

### 2.2 中国大陆网络加速（可选）

如果你在中国大陆，建议提前配置镜像源以加速安装：

**Git 镜像：**
```bash
git config --global url."https://mirror.ghproxy.com/https://github.com".insteadOf "https://github.com"
```

**pip 镜像：**
```bash
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
pip config set install.trusted-host mirrors.aliyun.com
```

**npm 镜像：**
```bash
npm config set registry https://mirrors.cloud.tencent.com/npm/ --global
```

## 三、快速安装（推荐）

这是最简单的安装方式，一键脚本自动处理所有依赖。

### 3.1 Linux / macOS / WSL2

打开终端，执行以下命令：

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

或者使用 wget：

```bash
wget -qO- https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

**中国大陆用户推荐使用中文社区镜像：**

```bash
curl -fsSL https://res1.hermesagent.org.cn/install.sh | bash
```

**脚本会自动完成以下操作：**
- 检测系统架构（x86_64 / arm64）
- 安装 uv、Python 3.11、Node.js v22
- 克隆 Hermes Agent 仓库
- 创建 Python 虚拟环境并安装依赖
- 注册 hermes 全局命令

**安装完成后，必须重新加载 shell 配置：**

```bash
# bash 用户
source ~/.bashrc

# zsh 用户（macOS 默认）
source ~/.zshrc
```

**验证安装：**
```bash
hermes --version
```

如果输出了版本号，说明安装成功！

### 3.2 Windows 原生 PowerShell

如果你不想安装 WSL2，可以直接使用 PowerShell 版本：

```powershell
irm https://res1.hermesagent.org.cn/install.ps1 | iex
```

安装完成后，关闭并重新打开 PowerShell，然后输入：

```powershell
hermes
```

## 四、手动安装（折腾版）

如果你想完全掌控安装过程，可以选择手动安装。

### 4.1 克隆仓库

```bash
git clone --recurse-submodules https://github.com/NousResearch/hermes-agent.git
cd hermes-agent
```

如果克隆时忘了 `--recurse-submodules`，补一下：

```bash
git submodule update --init --recursive
```

### 4.2 安装 uv 和虚拟环境

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
uv venv venv --python 3.11
```

### 4.3 安装依赖

```bash
export VIRTUAL_ENV="$(pwd)/venv"
uv pip install -e ".[all]"
```

**各模块说明：**

| 模块 | 功能 | 安装命令 |
|------|------|----------|
| 核心 | 基础功能 | `uv pip install -e "."` |
| messaging | Telegram/Discord 网关 | `uv pip install -e ".[messaging]"` |
| cron | 定时任务 | `uv pip install -e ".[cron]"` |
| voice | 麦克风+音频播放 | `uv pip install -e ".[voice]"` |
| mcp | MCP 支持 | `uv pip install -e ".[mcp]"` |
| slack | Slack 消息 | `uv pip install -e ".[slack]"` |
| honcho | AI 原生记忆 | `uv pip install -e ".[honcho]"` |

### 4.4 创建配置目录和文件

```bash
mkdir -p ~/.hermes/{cron,sessions,logs,memories,skills,pairing,hooks,image_cache,audio_cache,whatsapp/session}
cp cli-config.yaml.example ~/.hermes/config.yaml
touch ~/.hermes/.env
```

### 4.5 添加 API Key

编辑 `~/.hermes/.env` 文件，添加你的 API Key：

```bash
echo 'OPENROUTER_API_KEY=sk-or-v1-your-key-here' >> ~/.hermes/.env
```

### 4.6 添加到 PATH

```bash
mkdir -p ~/.local/bin
ln -sf "$(pwd)/venv/bin/hermes" ~/.local/bin/hermes

echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 4.7 验证安装

```bash
hermes version   # 检查命令是否可用
hermes doctor    # 运行诊断
hermes chat -q "Hello! What tools do you available?"  # 测试对话
```

## 五、Docker 安装（零依赖）

如果你不想安装 Python 环境，可以使用 Docker。

### 5.1 拉取镜像

```bash
docker pull nousresearch/hermes-agent:latest
```

### 5.2 交互式运行

```bash
docker run -it --rm \
  -v ~/.hermes:/home/hermes/.hermes \
  -v ~/.ssh:/home/hermes/.ssh:ro \
  -e OPENROUTER_API_KEY=sk-or-xxx \
  nousresearch/hermes-agent:latest
```

### 5.3 使用 Docker Compose

创建 `docker-compose.yml`：

```yaml
version: '3'
services:
  hermes:
    image: nousresearch/hermes-agent:latest
    volumes:
      - ~/.hermes:/home/hermes/.hermes
      - ~/.ssh:/home/hermes/.ssh:ro
    environment:
      - OPENROUTER_API_KEY=sk-or-xxx
    stdin_open: true
    tty: true
```

然后运行：

```bash
docker compose up -d
docker compose exec hermes hermes
```

## 六、配置 LLM 模型

安装好之后，需要配置 LLM 提供商。

### 6.1 使用交互式向导

```bash
hermes setup
```

这会启动一个交互式向导，引导你完成：
- 选择模型提供商
- 输入 API Key
- 基本偏好设置

### 6.2 常用模型提供商

| 提供商 | 配置方式 | 说明 |
|--------|----------|------|
| Nous Portal | OAuth 登录，零配置 | 官方订阅服务，最简单 |
| Anthropic Claude | API Key 或 Claude Code 授权 | 注意：目前尚未被封禁 |
| OpenRouter | API Key | 支持 200+ 模型，灵活度高 |
| DeepSeek | API Key | 国内用户友好，性价比高 |
| 阿里云 DashScope | API Key | 支持 Qwen 系列 |
| Ollama | 本地运行 | 完全离线，隐私优先 |
| 自定义端点 | OpenAI 兼容 API | vLLM、SGLang 等 |

### 6.3 配置示例

**使用 DeepSeek：**
```bash
hermes config set model deepseek/deepseek-chat
hermes config set DEEPSEEK_API_KEY sk-your-key-here
```

**使用 Ollama 本地模型：**
```bash
# 先启动 Ollama
ollama serve

# 配置 Hermes 使用本地模型
hermes config set model ollama/llama3
```

## 七、常见问题排查

### 7.1 hermes: command not found

**原因：** 没有重新加载 shell 配置，或 PATH 配置有误。

**解决：**
```bash
# 重新加载 shell
source ~/.bashrc  # 或 source ~/.zshrc

# 检查 PATH
echo $PATH

# 手动添加 PATH
export PATH="$HOME/.local/bin:$PATH"
```

### 7.2 API key not set

**原因：** 没有配置模型提供商。

**解决：**
```bash
# 运行模型配置向导
hermes model

# 或直接设置
hermes config set OPENROUTER_API_KEY your_key
```

### 7.3 curl 命令失败

**原因：** 网络问题或 curl 未安装。

**解决：**
```bash
# 检查网络是否能访问 GitHub
ping github.com

# 设置代理（如果需要）
export https_proxy=http://your-proxy:port

# 安装 curl（Linux）
sudo apt install curl  # Ubuntu/Debian
brew install curl      # macOS
```

### 7.4 Python 相关错误

**原因：** Python 版本不兼容或依赖安装失败。

**解决：**
```bash
# 运行诊断工具
hermes doctor

# 检查 Python 版本
python3 --version
```

## 八、开始使用

配置好后，直接输入：

```bash
hermes
```

即可进入全交互式终端界面，开始和 Agent 对话。

**常用命令：**

| 命令 | 说明 |
|------|------|
| `hermes` | 启动交互式 CLI 聊天 |
| `hermes setup` | 运行完整设置向导 |
| `hermes model` | 切换模型提供商和模型 |
| `hermes tools` | 配置启用的工具 |
| `hermes gateway` | 启动消息网关（Telegram 等） |
| `hermes update` | 更新到最新版本 |
| `hermes doctor` | 诊断问题 |
