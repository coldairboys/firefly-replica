# 相册页面配置指南

这个项目自带相册功能，但默认不一定开启。

如果你想把图片单独做成“相册页”，而不是只塞进文章里，这篇就很适合你。

## 一、相册功能开关在哪

先看：

```text
src/config/siteConfig.ts
```

你要确保这里打开：

```typescript
pages: {
  gallery: true,
}
```

如果这里是 `false`，即使你配了相册内容，页面也不会正常作为站点功能开放。

## 二、相册配置文件在哪

```text
src/config/galleryConfig.ts
```

你现在这个项目里已经有一个示例相册。

## 三、相册图片应该放哪里

```text
public/gallery/
```

每一个相册都对应这里的一个子目录。

比如配置里写：

```typescript
id: "firefly-2026"
```

那么图片目录就应该是：

```text
public/gallery/firefly-2026/
```

## 四、怎么添加一个相册

在 `galleryConfig.ts` 里加一个数组项即可：

```typescript
albums: [
  {
    id: "my-trip-2026",
    name: "我的旅行相册",
    description: "记录一次旅行中的照片",
    location: "杭州",
    date: "2026-04-23",
    tags: ["旅行", "摄影"],
  },
]
```

然后在下面这个目录里放图片：

```text
public/gallery/my-trip-2026/
```

## 五、封面图规则怎么理解

有 3 种情况：

### 情况 1：你手动写了 `cover`

```typescript
cover: "cover.avif"
```

那它会优先用你指定的封面。

### 情况 2：你没写 `cover`，但目录里有 `cover.*`

比如：

```text
cover.avif
cover.webp
cover.jpg
```

系统会优先把这个当封面。

### 情况 3：前两种都没有

系统通常会用第一张图片当封面。

## 六、支持哪些图片格式

常见支持：

- jpg
- png
- webp
- avif
- gif

如果你只是日常用，推荐优先用：

- `webp`
- `avif`

因为体积更小。

## 七、访问地址怎么理解

### 相册总页

```text
/gallery/
```

### 单个相册页

```text
/gallery/相册id/
```

比如：

```text
/gallery/my-trip-2026/
```

## 八、列数和瀑布流怎么调

在 `galleryConfig.ts` 里：

```typescript
columnWidth: 240
```

这个值的意思可以简单理解成：

- 数值越小，同屏列数越多
- 数值越大，同屏列数越少

如果你觉得图片太挤，就把它调大一点。

## 九、新手最稳的添加步骤

建议按这个顺序做：

1. 先在 `siteConfig.ts` 打开 `gallery: true`
2. 在 `galleryConfig.ts` 新增一个相册
3. 去 `public/gallery/相册id/` 放图片
4. 刷新本地页面检查
5. 没问题再推送

## 十、常见问题

### Q: 相册页打不开？

先检查：

1. `pages.gallery` 有没有开启
2. `galleryConfig.ts` 有没有写对
3. 相册 `id` 和目录名是不是一致

### Q: 图片不显示？

通常是因为：

1. 图片没放到 `public/gallery/对应目录/`
2. 文件名写错
3. 图片格式不对

### Q: 封面图不对？

优先检查：

1. 有没有手动写 `cover`
2. 目录里有没有 `cover.*`
3. 第一张图片是不是你想要的那张

## 最后总结

相册功能其实很好记：

- 相册信息写在 `galleryConfig.ts`
- 相册图片放在 `public/gallery/`
- 每个相册一个目录

如果你后面想把博客做得更完整，这个功能非常值得补上。
