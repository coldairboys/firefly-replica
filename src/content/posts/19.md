# 备份和恢复指南

## 基本概念

备份是保护博客数据的重要措施，可以：
- 防止数据丢失
- 快速恢复网站
- 迁移到新服务器
- 回滚到之前的版本

### 2. 手动备份

**步骤：**
1. 压缩项目目录
2. 存储到安全位置
3. 定期更新备份

**命令：**

```bash
# Windows
Compress-Archive -Path . -DestinationPath blog-backup-$(Get-Date -Format "yyyy-MM-dd").zip

# Linux/macOS
zip -r blog-backup-$(date +%Y-%m-%d).zip . -x "node_modules/*" "dist/*" ".git/*"
```

### 3. 自动化备份

**使用 GitHub Actions：**

```yaml
name: Backup Blog

on:
  schedule:
    - cron: '0 0 * * *'  # 每天凌晨执行
  push:
    branches:
      - main

jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          fetch-depth: 0
      
      - name: Create backup
        run: |
          zip -r blog-backup-$(date +%Y-%m-%d).zip . -x "node_modules/*" "dist/*" ".git/*"
      
      - name: Upload backup to GitHub Releases
        uses: softprops/action-gh-release@v1
        with:
          files: blog-backup-*.zip
          name: Backup ${{ github.ref_name }} $(date +%Y-%m-%d)
          tag_name: backup-$(date +%Y-%m-%d)
          body: Automated backup
```

### 4. 云存储备份

**使用 rclone：**

```bash
# 安装 rclone
curl https://rclone.org/install.sh | sudo bash

# 配置 rclone
rclone config

# 同步备份
rclone sync ./ blog-backup:blog-backup/$(date +%Y-%m-%d)
```

## 恢复方法

### 1. 从 Git 恢复

**步骤：**
1. 克隆仓库
2. 安装依赖
3. 构建网站

```bash
# 克隆仓库
git clone https://github.com/yourusername/your-blog.git

# 进入目录
cd your-blog

# 安装依赖
npm install

# 构建网站
npm run build

# 本地预览
npm run preview
```

### 2. 从备份文件恢复

**步骤：**
1. 解压备份文件
2. 安装依赖
3. 构建网站

```bash
# 解压备份
unzip blog-backup-2026-04-24.zip

# 进入目录
cd blog-backup-2026-04-24

# 安装依赖
npm install

# 构建网站
npm run build
```

### 3. 从云存储恢复

**步骤：**
1. 下载备份
2. 解压文件
3. 安装依赖
4. 构建网站

```bash
# 从云存储下载
rclone copy blog-backup:blog-backup/2026-04-24 ./blog-backup

# 进入目录
cd blog-backup

# 安装依赖
npm install

# 构建网站
npm run build
```

## 灾难恢复

### 1. 紧急恢复流程

1. **评估损失**：确定哪些文件受损
2. **选择备份**：选择最近的完整备份
3. **恢复文件**：从备份中恢复受损文件
4. **验证功能**：测试网站功能
5. **预防措施**：加强备份策略

### 2. 常见灾难场景

| 场景 | 原因 | 解决方案 |
|------|------|----------|
| 代码错误 | 代码修改导致网站崩溃 | 回滚到之前的版本 |
| 服务器故障 | 服务器宕机或数据丢失 | 从备份恢复到新服务器 |
| 域名问题 | 域名过期或被抢注 | 备份网站内容，重新配置域名 |
| 安全漏洞 | 被黑客攻击 | 从备份恢复，修复漏洞 |
| 数据损坏 | 数据库或文件损坏 | 从备份恢复数据 |

## 高级配置

### 1. 自动化备份脚本

**Windows 脚本：**

```powershell
# backup.ps1
$date = Get-Date -Format "yyyy-MM-dd"
$backupDir = "D:\Backups\Blog"
$backupFile = "blog-backup-$date.zip"
$projectDir = "E:\GITHUB\firefly-replica"

# 创建备份目录
if (!(Test-Path $backupDir)) {
    New-Item -ItemType Directory -Path $backupDir -Force
}

# 压缩备份
Compress-Archive -Path $projectDir -DestinationPath "$backupDir\$backupFile" -CompressionLevel Optimal

# 保留最近30天的备份
Get-ChildItem $backupDir -Filter "*.zip" | 
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-30) } | 
    Remove-Item -Force

Write-Host "备份完成: $backupFile"
```

**Linux/macOS 脚本：**

```bash
#!/bin/bash
# backup.sh
date=$(date +%Y-%m-%d)
backup_dir="/backups/blog"
backup_file="blog-backup-$date.zip"
project_dir="/home/user/blog"

# 创建备份目录
mkdir -p $backup_dir

# 压缩备份
zip -r "$backup_dir/$backup_file" $project_dir -x "node_modules/*" "dist/*" ".git/*"

# 保留最近30天的备份
find $backup_dir -name "*.zip" -mtime +30 -delete

echo "备份完成: $backup_file"
```

