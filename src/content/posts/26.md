# SEO 与 RSS 订阅配置指南

这篇讲的是博客后期非常实用的一组能力：

- RSS 订阅
- robots.txt
- sitemap
- 页面 OG 信息

你可以把它们理解成：

- RSS：给读者订阅
- robots / sitemap：给搜索引擎理解你的网站
- OG：让分享链接时看起来更完整

## 一、RSS 是什么

RSS 可以让别人用阅读器订阅你的博客更新。

这个项目已经内置了 RSS 页面和 RSS XML 文件。

### 相关文件

```text
src/pages/rss.astro
src/pages/rss.xml.ts
```

### 访问地址

```text
/rss/
/rss.xml
```

## 二、RSS 里哪些文章会被收录

通常普通公开文章都会被收录。  
如果文章设置了密码保护，RSS 里一般不会直接输出完整内容。

所以密码文章和普通文章的表现不一样，这是正常的。

## 三、robots.txt 是什么

它是给搜索引擎爬虫看的一个说明文件。

项目里有相关文件：

```text
src/pages/robots.txt.ts
```

不过你这个仓库里同时还有：

```text
public/robots.txt
```

最终生效时，通常要以你实际部署输出的结果为准。

## 四、sitemap 是什么

sitemap 可以理解成“网站地图”，帮助搜索引擎知道你有哪些页面。

这个项目构建后会生成：

```text
sitemap-index.xml
```

这个文件对搜索引擎收录很有帮助。

## 五、OG 信息是什么

OG 就是你把文章链接发到社交平台时，显示出来的：

- 标题
- 描述
- 图片

这些信息主要在页面布局里生成：

```text
src/layouts/Layout.astro
```

## 六、SEO 最基础该改哪些地方

对于个人博客来说，优先把这几个改好就很够用：

### 1. 站点标题和描述

位置：

```text
src/config/siteConfig.ts
```

重点字段：

```typescript
title
description
keywords
site_url
```

### 2. 每篇文章的标题和简介

每篇文章最好都写：

```yaml
title: 文章标题
description: 一句话简介
```

### 3. 文章封面图

如果文章有封面图，分享出去通常也更完整。

## 七、RSS 对新手最实用的意义

很多新手会觉得 RSS 很“老”，其实它依然很有用。

尤其适合：

- 技术博客
- 学习笔记站
- 不想依赖平台推荐流量的人

## 八、建议你实际怎么做

### 第一步

先确认 `siteConfig.ts` 里的这些信息都写完整：

```typescript
title: "Coldairboy学习笔记"
site_url: "https://blogboy.eu.cc"
description: "Coldairboy 的个人博客样板，用来记录学习笔记与折腾记录。"
keywords: ["Coldairboy", "学习笔记", "Astro", "Firefly", "博客"]
```

### 第二步

每篇文章尽量补齐：

- `title`
- `description`
- `image`
- `published`

### 第三步

构建一次看看有没有正常生成：

```powershell
cd E:\GITHUB\firefly-replica
pnpm build
```

### 第四步

部署后访问这些地址检查：

```text
/rss/
/rss.xml
/sitemap-index.xml
```

## 九、常见问题

### Q: RSS 页面能打开，但订阅器读不到？

先检查：

1. `/rss.xml` 能不能直接访问
2. 域名是不是已经正确部署
3. 构建是不是成功过

### Q: 搜索引擎为什么还没收录？

这不一定是配置错了，也可能只是还没来得及抓取。

先做这些事：

1. 确保域名可正常访问
2. 确保 sitemap 可以访问
3. 页面标题和描述写完整
4. 持续更新内容

### Q: 分享链接为什么没有图片？

一般先检查：

1. 文章有没有封面图
2. 站点地址 `site_url` 是否正确
3. 主题是否开启了 OG 图相关配置

## 十、最后总结

如果你只想记住最关键的点，就记这几个：

- `siteConfig.ts` 决定站点基础 SEO 信息
- `/rss.xml` 是订阅地址
- `sitemap-index.xml` 对收录有帮助
- 每篇文章最好写完整标题、简介和封面

SEO 不用一上来就研究很深。  
先把这些基础项配好，你的博客就已经比“只会发文章”更完整了。
