【publish-blog.bat 使用说明】

1. 直接双击 publish-blog.bat
2. 脚本会先显示当前 Git 状态
3. 然后让你输入一次提交说明
4. 回车后会自动执行：
   - git add .
   - git commit -m "你的说明"
   - git push origin main
5. 执行结束后窗口不会闪退
6. 不管成功还是失败，都会生成一个结果文件：
   publish-result.txt

【如果你不想输入提交说明】

直接回车就行。
脚本会自动使用默认说明：
update blog content

【怎么判断有没有推送成功】

打开 publish-result.txt

如果里面出现：
Result: push success

就说明已经推送成功。

【如果失败了】

先看黑窗口里的英文报错。
常见原因：

1. 网络连不上 GitHub
2. 你没有新的改动
3. Git 登录或远程仓库有问题

【建议】

第一次运行时，建议不要双击。
先在终端里执行一次，方便观察：

cd E:\GITHUB\firefly-replica
.\publish-blog.bat
