publish-blog.bat 使用说明

1. 双击 publish-blog.bat
2. 它会自动执行下面三步：
git add .
git commit -m "你的说明"（只有真的有新改动时才会提交）
git push origin main（只要本地有提交没推上去，它就会继续推）
3. 运行结束后不会直接闪退
4. 结果会写入 publish-result.txt

什么时候用它
你刚改完博客内容，想一键提交并推送，就双击 publish-blog.bat

什么时候用 push-only.bat
你已经执行过 git commit，只是因为网络问题没推上去
这时双击 push-only.bat 就行，它只负责重新推送

怎么判断有没有成功
打开 publish-result.txt
如果看到 Result: push success，就说明已经成功推送到 GitHub

常见失败原因
1. 网络暂时连不上 GitHub
2. 代理或 VPN 设置有问题
3. GitHub 登录状态异常

终端手动运行方式
cd E:\GITHUB\firefly-replica
.\publish-blog.bat

如果你只是重推已经提交过的内容
cd E:\GITHUB\firefly-replica
.\push-only.bat
